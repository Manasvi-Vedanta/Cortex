Question 2
Kotlin Easy ✗
In Kotlin, which keyword is used to declare a mutable variable, meaning its value can be changed after initialization?
  • A val
  • B var
  • C const
  • D let
💡 Explanation

`var` is used for mutable variables, while `val` is used for read-only (immutable) variables in Kotlin.

Question 3
mobile operating systems Easy ✗
Which of the following is NOT primarily a mobile operating system?
  • A iOS
  • B Android
  • C watchOS
  • D macOS
💡 Explanation

iOS, Android, and watchOS are all mobile or wearable operating systems. macOS is Apple's desktop operating system.

Question 4
Java Easy ✗
In Java, which keyword is used to declare a variable whose value cannot be changed once assigned (a constant reference or immutable primitive)?
  • A static
  • B final
  • C const
  • D fixed
💡 Explanation

`final` is the keyword used in Java to declare a constant variable or to indicate that a method cannot be overridden or a class cannot be subclassed. `const` exists in C++ but not as a keyword for variable declaration in Java.

Retrofit Medium ✓
When using Retrofit for making network requests in an Android application, what is the primary role of an interface annotated with `@GET`, `@POST`, etc.?
  • A To define the application's user interface layout.
  • B To specify the HTTP methods and endpoints for API calls.
  • C To manage local database transactions.
  • D To handle asynchronous background processing tasks.
💡 Explanation

Retrofit uses annotated interfaces to declare how HTTP requests will be made, mapping Java/Kotlin methods to specific HTTP verbs (GET, POST, PUT, DELETE) and defining endpoint paths.

Question 6
Jetpack Compose Medium ✗
In Jetpack Compose, which composable is specifically designed for efficiently displaying a long, scrolling list of items, only rendering the visible ones?
  • A Column
  • B Row
  • C LazyColumn
  • D Box
💡 Explanation

`LazyColumn` (and `LazyRow`) are the Compose equivalents of `RecyclerView`, designed to efficiently render large numbers of items by only composing and laying out the items currently visible on screen.

Question 7
Gradle Medium ✗
What is the main function of Gradle in the context of Android application development?
  • A It's an integrated development environment (IDE) for coding Android apps.
  • B It's a build automation system used for compiling code, managing dependencies, and packaging the application.
  • C It's a database management system for storing application data locally.
  • D It's a framework for creating cross-platform mobile applications using JavaScript.
💡 Explanation

Gradle is the primary build system used by Android Studio, responsible for automating tasks like compiling source code, linking libraries, creating the APK/AAB, and managing project dependencies.

Question 8
Coroutines Hard ✗
In Kotlin Coroutines, what is the key conceptual difference between using `launch` and `async` to start a coroutine?
  • A `launch` always runs on the main thread, while `async` runs on a background thread.
  • B `launch` is used for fire-and-forget tasks and returns a `Job`, whereas `async` is used for tasks that return a result and returns a `Deferred`.
  • C `async` is a blocking call, while `launch` is non-blocking.
  • D They are functionally identical; the choice depends purely on coding style preference.
💡 Explanation

`launch` is used when you don't need a result back from the coroutine; it returns a `Job` for managing its lifecycle. `async` is used when the coroutine needs to compute a value that will be awaited later; it returns a `Deferred`, which is a cancellable future that provides the result.

Question 9
Room Database Hard ✗
In Android's Room Persistence Library, which component is typically an abstract class annotated with `@Database` and serves as the main access point to the underlying SQLite database?
  • A Entity
  • B DAO (Data Access Object)
  • C TypeConverter
  • D RoomDatabase
💡 Explanation

The `RoomDatabase` class is the main database holder and acts as the primary access point for your app's persisted data. It typically contains abstract methods that return DAOs, which then provide methods for interacting with the database tables (Entities).

Question 10
Firebase Hard ✗
A mobile application requires a scalable, serverless backend to store user data, enable real-time synchronization across devices, and manage user authentication. Which combination of Firebase services would best meet these requirements?
  • A Firebase Hosting for data storage, Firebase Cloud Messaging for real-time updates, and Firebase Remote Config for authentication.
  • B Cloud Firestore for data storage and real-time sync, Firebase Authentication for user management, and Firebase Cloud Functions for custom backend logic.
  • C Firebase Storage for all data, Firebase Performance Monitoring for real-time sync, and Firebase A/B Testing for authentication.
  • D Google Analytics for Firebase for data storage, Firebase Dynamic Links for authentication, and Firebase App Distribution for real-time sync.
💡 Explanation

Cloud Firestore is a flexible, scalable NoSQL cloud database for mobile, web, and server development that keeps your data in sync across client apps in real-time. Firebase Authentication provides backend services for easy-to-use, secure authentication. Firebase Cloud Functions can extend the backend with custom logic when needed.
