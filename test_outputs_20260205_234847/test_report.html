Jetpack Compose Medium ✓
In Jetpack Compose, what is the primary role of a @Composable function?

  • B To compile Kotlin code into native C++
  • C To define database schemas for Room
  • D To perform network requests on the main thread

Mvvm Architecture Medium ✓

  • A ViewModel
  • B Repository
  • C Database (Room)
  • D OkHttp Interceptor
💡 Explanation

The ViewModel holds and manages UI-related data/state and survives configuration changes, exposing state to the View.

Question 7
Rest Apis (Retrofit/Okhttp) Medium ✓
You need to call a REST API using Retrofit and parse JSON into Kotlin objects. Which combination is most appropriate?
  • A Retrofit + OkHttp + a converter (e.g., Moshi/Gson)
  • B Room + LiveData + ViewBinding
  • C Jetpack Compose + Navigation + DataStore only
  • D Gradle + ProGuard + ADB only
💡 Explanation

Retrofit uses OkHttp for HTTP calls and a converter (like Moshi/Gson) to map JSON to Kotlin data classes.

Question 8
Coroutines Hard ✗
A Compose screen triggers a network request when a button is clicked. The request must be cancellable when the user leaves the screen. Where should you launch the coroutine to best match lifecycle and cancellation needs?
  • A In the ViewModel using viewModelScope
  • B In a global scope (GlobalScope.launch) so it always finishes
  • C On the main thread without coroutines
  • D Inside an OkHttp interceptor
💡 Explanation

Launching in viewModelScope ties the work to the ViewModel lifecycle, enabling structured concurrency and automatic cancellation when the ViewModel is cleared.

Question 9
Android Jetpack (Architecture Components) Hard ✗
Your app must work offline and sync when connectivity returns. Which Android Jetpack Architecture Components-based approach best supports this requirement?
  • A Repository pattern with Room as a local source of truth and a sync layer (e.g., WorkManager) to update from network
  • B Store all data only in memory inside the Activity to avoid disk I/O
  • C Make every screen fetch directly from Retrofit and disable caching
  • D Use only SharedPreferences for all relational data and run sync on the main thread
💡 Explanation

A repository mediates between Room and network; Room provides offline persistence, and WorkManager can schedule reliable background sync.
